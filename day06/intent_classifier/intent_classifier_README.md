# Day 06：规则式意图分类器

## 项目简介

这是 Python 学习计划 Week 1 · Day 6 的小项目。

项目目标是综合运用前五天学习的内容，完成一个简单的命令行意图分类器。

程序接收用户输入的一句话，并将其分类为以下三种意图之一：

* 学习
* 查询
* 闲聊

当前版本采用关键词匹配规则，不使用外部分词库、正则表达式或大语言模型。

## 第一阶段：关键词库设计

### 设计原则

#### 1. 优先选择强特征词

关键词出现后，应当能够较明确地表示用户意图。

例如：

* “复习”通常表示学习意图
* “搜索”通常表示查询意图
* “你好”通常表示闲聊意图

#### 2. 避免使用过短关键词

当前程序采用子串匹配：

```python
keyword in text
```

因此不建议直接使用“学”“查”等单字。

例如：

```text
学校附近有什么？
```

如果将“学”作为学习关键词，程序可能因为“学校”中包含“学”，错误地判断为学习意图。

#### 3. 控制词库规模

每种意图先设置 5～8 个关键词。

当前阶段不追求覆盖所有表达，而是先完成一个可运行、可测试的最小版本。

#### 4. 使用列表保存关键词

关键词不直接重复写在判断语句中，而是分别保存在列表中，方便后续修改和扩展。

## 文本清洗规则

当前版本只进行两项清洗：

1. 使用 `strip()` 删除首尾空白；
2. 使用 `lower()` 将英文字母转换成小写。

示例：

```python
cleaned_text = text.strip().lower()
```

输入：

```text
  我想学习 PYTHON  
```

清洗后：

```text
我想学习 python
```

当前版本暂不处理：

* 中文分词
* 标点符号清除
* 全角与半角字符转换
* 错别字修正
* 同义词识别
* 否定结构
* 语义理解

## 匹配算法

当前版本采用子串包含匹配：

```python
keyword in text
```

例如：

```python
text = "我想学习 python"

print("学习" in text)
```

输出：

```text
True
```

关键词列表中的任意一个关键词出现，即认为该类别被命中。

预计使用：

```python
for keyword in keywords:
    if keyword in text:
        return True

return False
```

## 分类优先级

分类顺序设置为：

1. 学习
2. 查询
3. 闲聊
4. 默认返回闲聊

流程如下：

```text
输入文本
   ↓
是否命中学习关键词？
   ├── 是 → 返回“学习”
   └── 否
        ↓
是否命中查询关键词？
   ├── 是 → 返回“查询”
   └── 否
        ↓
是否命中闲聊关键词？
   ├── 是 → 返回“闲聊”
   └── 否 → 默认返回“闲聊”
```

学习意图优先于查询意图，并不是因为学习意图更加重要，而是因为学习关键词通常更加具体。

例如：

```text
如何高效复习？
```

这句话既具有疑问特征，也包含“复习”。

“复习”比“如何”更加具体，因此应当优先分类为学习。

---

## 默认分类说明

当用户输入没有命中任何关键词时，当前版本默认返回：

```text
闲聊
```

这并不表示程序真正理解了用户正在闲聊。

它只表示：

```text
当前规则没有发现明确的学习或查询意图。
```

在后续版本中，可以增加：

```text
未知
```

或：

```text
其他
```

作为独立类别。

但为了满足 Day 6 只有三种意图的要求，本版本使用闲聊作为默认结果。

## 空输入处理

空输入不属于任何意图。

因此，空输入应当由命令行交互层处理，而不是直接交给分类函数。

示例：

```python
user_input = input("请输入一句话：")
cleaned_text = user_input.strip()

if not cleaned_text:
    print("输入不能为空，请重新输入。")
    continue
```

需要注意：

```python
not cleaned_text
```

表示字符串为空。

空字符串：

```python
""
```

在布尔判断中会被视为：

```python
False
```

## 持续交互

程序使用：

```python
while True:
```

持续接收用户输入。

基本结构：

```python
while True:
    user_input = input("请输入一句话：")

    if user_input == "退出指令":
        break

    intent = classify_intent(user_input)
    print(f"→ 意图类别：{intent}")
```

---

## 退出指令

以下输入用于退出程序：

```text
exit
quit
q
```

退出判断应放在意图分类之前。

示例：

```python
if cleaned_text in ["exit", "quit", "q"]:
    print("程序已退出。")
    break
```

不能将退出指令交给意图分类器，否则它们可能被错误地分类为闲聊。

## 已知缺陷

### 1. 无法理解否定结构

例如：

```text
我不想学习，只想聊天。
```

程序可能检测到“学习”，然后立即返回：

```text
学习
```

程序并不知道“不想”否定了后面的“学习”。

### 2. 子串匹配可能误判

例如：

```text
我正在检查作业。
```

如果使用“查”作为查询关键词，可能错误命中查询意图。

因此当前词库避免使用单字关键词。

### 3. 无法识别词库外的同义词

例如：

```text
帮我搜一下资料。
```

如果词库中只有“搜索”，没有“搜”，程序可能无法识别。

### 4. 多种意图只能选择一种

例如：

```text
先教我 Python，然后查询今天的天气。
```

这句话同时包含学习和查询意图。

当前程序会根据固定优先级，只返回：

```text
学习
```

### 5. 默认闲聊的可信度有限

例如：

```text
帮我订一张机票。
```

这句话不一定是闲聊，但如果没有命中学习或查询关键词，当前版本会默认返回闲聊。

## 当前项目定位

本项目不是完整的自然语言理解系统。

它的主要价值是练习如何将多个基础知识组合成一个完整程序：

```text
字符串处理
    +
列表
    +
循环
    +
条件判断
    +
函数
    +
命令行交互
    +
自动测试
```

项目重点不是让分类器达到很高的准确率，而是确保代码：

* 能运行
* 能解释
* 能测试
* 能发现误判
* 能继续重构
