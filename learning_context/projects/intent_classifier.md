# 规则式意图分类器

## 项目定位

这是 Day 06 的 Python 基础综合练习。程序使用关键词匹配，将命令行输入分为“学习、查询、闲聊”。

## 已有证据

- 主实现：`day06/intent_classifier/`。
- 使用了字符串处理、列表、条件判断、函数、循环和简单测试。
- `misjudge_analysis.md` 记录了已知的误判与局限。

## 已知限制

- 不能理解否定结构。
- 子串匹配可能误判。
- 多意图输入只能按固定优先级返回一个类别。
- 默认“闲聊”不等于真正理解输入。

## 建议的后续回访点

- 学习字典后，用字典管理不同意图的关键词。
- 学习 JSON 和文件读取后，将关键词配置与分类逻辑分离。
- 学习异常处理后，为配置读取增加缺失文件和格式错误提示。
- 学习正则表达式后，再尝试处理简单否定结构，并同时补充测试。
